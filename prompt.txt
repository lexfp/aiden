## Overview

Dogs vs Cats Tic-Tac-Toe is a funny, browser-based two-player game that replaces the classic X
and O pieces with animated dog and cat emoji. Two players take turns placing their animal on a
3x3 grid. The first player to get three of their animal in a row wins — and is rewarded with an
over-the-top fart victory celebration. The game is delivered as a single self-contained
index.html file with all CSS and JavaScript inlined — no build tools, no server, no external
dependencies.

## UI / Behavior

Build a single-screen layout with this structure top to bottom:

1. Title: "🐶 Dogs vs Cats 🐱" — large, playful font. Comic Sans or similar encouraged.
2. Turn Indicator: "🐶 Dog's turn!" or "🐱 Cat's turn!" — updates immediately after every move.
3. 3x3 Grid: Large clearly-bordered square cells. Empty cells show a subtle color change on
   hover. Occupied cells show the animal emoji centered at ~3rem font size with no hover effect.
   Winning cells pulse with a glowing green border and yellow background (hand-written keyframe).
4. Play Again 💨 Button: Always visible below the board. Resets all state on click.

Victory Overlay (shown immediately on win, blocks the board):
- Semi-transparent dark overlay covering the full board area
- Winning animal emoji in giant size, CSS bounce or spin animation (hand-written keyframe)
- Winner text in oversized font: "🐶 DOG WINS! 💨💨💨" or "🐱 CAT WINS! 💨💨💨"
- Multiple 💨 emoji drifting upward via staggered CSS keyframe animations (hand-written)
- Web Audio API fart sound fires immediately
- After 3 seconds: "Play Again 💨" button pulses to invite a rematch

Draw State: No overlay. Turn indicator changes to "😐 It's a draw! No one farted."
Play Again button pulses.

## Testing

This is a single HTML file with no test framework. Perform manual verification and log every
result to output.txt under a TEST entry.

Happy Path Tests:
1. Start game → Player 1 (dog) places first → turn switches to cat
2. Play through to a dog win (top row) → fart sound plays → overlay appears → winning cells highlighted
3. Play through to a cat win (diagonal) → fart sound plays → cat overlay appears
4. Fill the board with no winner → draw message appears, no fart sound
5. Click "Play Again 💨" → board resets → dog's turn shown → can play again

Edge Case Tests:
6. Click an occupied cell → nothing happens, no piece placed, no turn change
7. Click any cell during celebration overlay → nothing happens (clicks blocked)
8. Open in mobile viewport (or DevTools mobile emulation, 375px wide) → cells are tappable at 80px+ each

Autoplay Test:
9. Open in a fresh browser tab → confirm fart sound plays on win OR fallback 💨 button appears

Log PASS or FAIL for each test to output.txt. If any test fails, fix and re-run before pushing.
